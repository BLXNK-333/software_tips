Python Tips

Установка виртуального окружения

Установка виртуального окружения python с дублированием файлов, запускать из каталога проекта:

python3 -m venv .venv

если нужно установить зависимости, временно отключаем переменные proxy

source .venv/bin/activate
unset all_proxy && unset ALL_PROXY
pip install --upgrade pip

далее можно влить requirements.txt или другие пакеты.

Установка виртуального окружения с передыдущей версией Python:

Чтобы посмотреть версии установленных Python:

ls -la /usr/bin/python*

Для установки, например Python 3.12, создайте виртуальное окружение с нужной версией:

python3.12 -m venv .venv

Создание и использование файла requirements.txt для управления зависимостями

Команда для создания файла requirements.txt, который содержит список всех установленных пакетов Python и их версий, чтобы в дальнейшем это можно было использовать для восстановления среды Python с такими же зависимостями.

pip freeze > requirements.txt

При выполнении этой команды pip проходит по списку пакетов в файле requirements.txt и устанавливает каждый из них (если они еще не установлены) с учетом указанных версий.

pip install -r requirements.txt

Активация виртуальной среды

Активация виртуальной среды Python, расположенной по указанному пути:

source /.../.venv/bin/activate

deactivate - деактивация

Устранение ошибок, связанных с прокси, путем установки переменных окружения

Чтобы убрать ошибки связанные с proxy, при вызове программы из консоли, добавить в __init__.py:

import os

# Установка переменных окружения
os.environ["ALL_PROXY"] = ""
os.environ["NO_PROXY"] = ""
os.environ["all_proxy"] = ""
os.environ["no_proxy"] = ""

Пример файловой структуры:

~/project/
├── 📁 .git/
├── 📁 .venv/
├── 📁 src/
│   ├── __init__.py  # вот сюда
│   ├── module_1.py
│   └── module_2.py
├── main.py
├── requirements.txt
├── .flake8
└── .gitignore

Также, можно реализовать с помощью библиотеки dotenv.

Временная точка входа в программу

Если в вашем модуле, используются импорты из модулей, стоящих выше в файловой иерархии. И вы хотите сделать там временную точку входа для тестирования или других целей, вам нужно добавить родительскую директорию импортируемого модуля в путь поиска модулей Python (sys.path). Это позволяет Python находить и импортировать модули, расположенные выше в иерархии директорий. Также, следует учесть, что в исполняемом файле - точке входа в проект, не может, по умолчанию, быть относительных импортов - только абсолютные.

Пример файловой структуры:

~/project/
├── 📁 .git/
├── 📁 .venv/
├── 📁 src/
│   ├── __init__.py  
│   ├── module_2.py  # допустим здесь будет временная точка входа в программу.
│   └── module_3.py
├── module_1.py  # из этого модуля нужен импорт в module2.py
├── requirements.txt
├── .flake8
└── .gitignore

Добавить в самое начало module2.py

import os
import sys

sys.path.insert(0, os.path.join(os.path.dirname(__file__), '..'))

Использовать, только как временное решение, напимер для тестирования. Этот костыль, также не будет работать, если в коде выстроена структура относительных импортов, так что лучше его вообще не использовать.

Количество строк в проекте

Если вам нужно рекурсивно подсчитать количество строк кода во всех файлах в поддиректориях проекта:

find . -type d \( -path './.venv' -o -path './.idea' -o -path './.git' \) -prune -o -name '*.py' -type f -exec wc -l {} + | awk '$1 != 0'

Использование виртуального окружения с sudo

При использовании sudo, виртуальное окружение не активируется автоматически. Вы можете попытаться запустить скрипт внутри активированного виртуального окружения, чтобы sudo использовал Python и библиотеки из этого окружения.

Для этого используйте команду sudo с переменной окружения PYTHONPATH, указывающей на виртуальное окружение:

sudo PYTHONPATH=$(python3 -c 'import site; print(site.getsitepackages()[0])') python3 example.py

Форматирует текущее время в строку

Позволяет получить строку с текущим временем для названия файла, включающую миллисекунды и разделенную тире.

from datetime import datetime

# Получаем текущее время
now = datetime.now()

# Форматируем время в строку для названия файла с миллисекундами
file_name = now.strftime("%Y-%m-%d_%H-%M-%S-%f")

print(file_name)  # Пример вывода: 2024-09-01_14-25-36-123456
