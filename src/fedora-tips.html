Изменение раскладки:

Используется для установки сочетания клавиш для переключения между источниками ввода (языками) в GNOME Desktop на Linux. В данном случае сочетание клавиш установлено на <Alt>Shift_L. 

gsettings set org.gnome.desktop.wm.keybindings switch-input-source "['<Alt>Shift_L']"

Для сброса к дефолтным:

gsettings set org.gnome.desktop.wm.keybindings switch-input-source "['<Super>space']"

Используется для загрузки и автоматической установки иконок Papirus на вашем Linux-устройстве.

wget -qO- https://git.io/papirus-icon-theme-install | sh

Разархивировать файлы из архива .tar / .tar.gz:

tar -xvf archive.tar directory/

tar -xzvf archive.tar.gz directory/

Заархивировать файлы или директории .tar / .tar.gz:

tar -cvf archive.tar file1 file2

tar -czvf archive.tar.gz directory_name

Список опций:

  • -x: Указывающая на операцию извлечения файлов из архива.
  • -v: Включающая подробный вывод.
  • -z: Которая указывает tar на использование gzip для распаковки архива.
  • -c: Которая создает новый архив.
  • -f archive.tar: Указывающая на файл архива, который нужно использовать.
  • directory/: Путь к файлу или директории в архиве, который нужно извлечь.

Выведет список всех установленных пакетов ядра Linux. Опция -qa означает "query all" (запросить все).

rpm -qa kernel

или более быстрый вариант

find /boot/vmli*

Найдет все файлы и директории, имена которых содержат "file-name".

find . -iname "*file-name*"

find ./ -type d -iname "directory"

Список опций:

  • ./: Это текущая рабочая директория, от которой начинается поиск.
  • -iname: Фильтр имени, позволяет выполнить поиск без учета регистра символов.
  • -type d: Указывает find искать только директории.

Найдет все строки в файлах, которые содержат текст "Nordic-darker" в указанной директории и её поддиректориях, игнорируя регистр символов, и выведет номера строк, где найдены совпадения.

grep -Irn "Nordic-darker" /path/to/directory

Список опций:

  • -I: Игнорировать бинарные файлы.
  • -r: Рекурсивный обход всех подкаталогов.
  • -n: Выводит номера строк.

Изменяет владельца каталога /path/to/directory и всех его содержимых файлов и подкаталогов на пользователя john.

sudo chown -Rv john /path/to/directory

Список опций:

  • -R: Рекурсивно изменяет владельца указанного файла или каталога, включая все файлы и подкаталоги внутри него.
  • -v: Выводит подробную информацию о действиях, выполняемых командой.

Чтобы рекурсивно изменить права доступа на drwxr-xr-x для данной директории и всех её подкаталогов

chmod -R 755 directory_name

Список опций:

  • -R: Рекурсивно изменяет права, включая все файлы и подкаталоги внутри него.
  • 755: Этот код соответствует правам drwxr-xr-x, где 7 означает, что владелец имеет полные права (rwx), а группа и другие пользователи имеют права только на чтение и выполнение (r-x).
  • 700: Владелец имеет полные права на чтение, запись и выполнение, а остальные пользователи не имеют никаких прав.
  • 644: Владелец имеет полные права на чтение и запись, а группа и остальные пользователи имеют только права на чтение.
  • 777: Все пользователи имеют полные права на чтение, запись и выполнение.

Плагин dnf-plugin-versionlock позволяет заблокировать версии пакетов, чтобы предотвратить их автоматическое обновление через dnf. Это может быть полезно, если вам нужна стабильная версия пакета, и вы не хотите, чтобы она обновлялась до более новой версии.

Для установки, выполните следующую команду:

sudo dnf install python3-dnf-plugin-versionlock

Добавить пакеты для заморозки, на примере драйверов nvidia:

sudo dnf versionlock add akmod-nvidia-470xx.x86_64 nvidia-gpu-firmware.noarch nvidia-settings-470xx.x86_64 xorg-x11-drv-nvidia-470xx.x86_64 xorg-x11-drv-nvidia-470xx-kmodsrc.x86_64 xorg-x11-drv-nvidia-470xx-libs.x86_64

Очистить список заблокированных пакетов:

sudo dnf versionlock clear

Настройка прокификации через tor на Fedora 40.

  1. Подключаем репозиторий Tor (В РФ TOR блокируют, поэтому пункт 1 пропускаем, потому что сервер с файлами не будет доступен):

    2. sudo nano /etc/yum.repos.d/tor.repo

    Вставить в файл и сохранить:

    [tor]
name=Tor for Fedora $releasever - $basearch
baseurl=https://rpm.torproject.org/fedora/$releasever/$basearch
enabled=1
gpgcheck=1
gpgkey=https://rpm.torproject.org/fedora/public_gpg.key
cost=100

  2. Устанавливаем пакеты Tor:

    3. sudo dnf install tor

  3. Посещаем сайт, выбираем опцию "None" и получаем список мостов:

    4. https://bridges.torproject.org/options?lang=en

  4. Открываем файл torrc в текстовом редакторе и добавляем указанные строки с настройками мостов и портов.

    5. sudo nano /etc/tor/torrc

    Пример конфига:

    # Configuration:
UseBridges 1
SOCKSPort 9050
ExcludeExitNodes {RU},{BY}
StrictNodes 1

# Увеличиваем время жизни цепочек, так как используем
# этот tor proxy, только для обхода блокировок.
ExitNodes {DE}
MaxCircuitDirtiness 14400
TrackHostExits *.chatgpt.com *.google.com *.youtube.com *stackoverflow.com
TrackHostExitsExpire 14400

# Vanilla Bridges:
bridge 23.88.56.229:18680 0131800F17ACED0D6CF979F3D0910D63E5F6015A
bridge 24.96.77.137:43729 3FF3DA81DCD859CD6B5D31D719ADD7E39882A404

  5. Перезапускаем службу Tor:

    6. sudo systemctl restart tor.service

  6. Проверяем журнал, чтобы убедиться, что он успешно загрузился и запустился:

    7. journalctl -exft Tor | grep Bootstrapped

    Должно быть так:

    Bootstrapped 100% (done): Done

    выйти ctrl + с

  7. Для включения автозапуска службы Tor при старте системы:

    8. sudo systemctl enable tor.service

Чтобы использовать Tor в качестве прокси-сервера, укажите следующие настройки в своем приложении для обозначенного типа прокси:

Type: Socks5
Host: 127.0.0.1
Port: 9050

Если используете браузер, чтобы предотвратить утечку DNS, нужно пустить DNS запросы через "мост". Поэтому включаем опцию, вроде этой Proxy DNS when using SOCKS v5, в Network Settings. Для проверки утечки DNS: 

https://dnsleaktest.com/

Также, чтобы наверняка понимать, что происходит можно использовать сниффер сетевого траффика Wireshark.

Использование Tor-browser и службы tor.service.

Для использования Tor-browser, нужно отключить службу tor.

  1. Перед запуском браузера отключите службу tor, т.к. по умолчанию, они используют одинаковые порты:

    2. sudo systemctl stop tor.service

  2. Посещаем сайт, выбираем опцию "None" и получаем список мостов:

    3. https://bridges.torproject.org/options?lang=en

  3. Запускаем Tor-browser и делаем следующее:

    4. Configure Connection > Add a bridge manually > Вставляем мосты > Connect

Если нужно снова запустить службу tor.service, закрыть браузер и выполнить команду:

sudo systemctl start tor.service

Работа с программами в фоне, из терминала.

  • Запустите текстовый редактор GNOME:
    • gnome-text-editor
  • Нажмите Ctrl + Z, чтобы приостановить выполнение программы и вернуть командный интерфейс.
  • Введите bg, чтобы переместить программу в фоновый режим, позволяя вам продолжать работу в терминале.
  • Если вам нужно вернуться к программе, введите fg, чтобы вернуть ее в основной режим.
  • Чтобы увидеть список программ, работающих в фоновом режиме, введите jobs.
  • Если вам нужно вернуть программу из фонового режима, введите fg %1, где %1 - это номер задачи из списка jobs.

Убиваем процессы в Linux.

Для отправки сигнала определенному процессу с указанным идентификатором (PID), используем команду со следующим синтаксисом:

kill -сигнал pid_процесса

Список опций:

  • -сигнал: Позволяет указать тип отправляемого сигнала (по умолчанию, параметр -сигнал равен -TERM).
  • pid_процесса: Идентификатор (PID) процесса, которому нужно отправить сигнал.

Пример:

blxnk@fedora: ps aux | grep "calculator"
blxnk       5534  0.2  2.6 1509032 154876 ?      Sl   08:16   0:02 /usr/bin/gnome-calculator
blxnk       7688  0.0  0.0 222564  2304 pts/0    S+   08:32   0:00 grep --color=auto --exclude-dir=.bzr --exclude-dir=CVS --exclude-dir=.git --exclude-dir=.hg --exclude-dir=.svn --exclude-dir=.idea --exclude-dir=.tox calculator

blxnk@fedora: kill -TERM 5534

Если сигнал -TERM не завершает, то в крайнем случае:

kill -KILL 5534

Чтобы узнать количество файлов в каталоге:

Замените:

Навигация по тексту:

  • Ctrl+A — Перемещает курсор в начало строки.
  • Ctrl+E — Перемещает курсор в конец строки.
  • Ctrl+Left Arrow — Перемещает курсор на одно слово влево.
  • Ctrl+Right Arrow — Перемещает курсор на одно слово вправо.

Редактирование текста:

  • Ctrl+U — Удаляет текст от курсора до начала строки.
  • Ctrl+K — Удаляет текст от курсора до конца строки.
  • Ctrl+W — Удаляет слово слева от курсора.
  • Ctrl+Y — Вставляет текст, который был удален последней командой удаления.
  • Ctrl+H — Удаляет один символ слева от курсора (эквивалентно Backspace).
  • Ctrl+D — Удаляет один символ под курсором или завершает сеанс, если введено в пустой строке.

Управление экраном:

  • Ctrl+L — Очищает экран (эквивалентно команде clear).
  • Shift+Page Up — Прокручивает экран вверх.
  • Shift+Page Down — Прокручивает экран вниз.

Работа с терминалом:

  • Ctrl+Shift+T — Открывает новую вкладку.
  • Ctrl+Shift+W — Закрывает текущую вкладку.
  • Ctrl+Shift+N — Открывает новое окно терминала.
  • Ctrl+Shift+Q — Закрывает окно терминала.
  • Ctrl+Shift+C — Копирует выделенный текст.
  • Ctrl+Shift+V — Вставляет скопированный текст.

Управление процессами:

  • Ctrl+C — Прерывает текущий процесс.
  • Ctrl+Z — Приостанавливает текущий процесс.
  • Ctrl+R — Поиск по истории команд.
  • Ctrl+D — Завершает сеанс (выйти из терминала), если введено в пустой строке.

Разрешение на выполнение команды без запроса пароля.

Пример:

Открыть файл /etc/sudoers.tmp в текстовом редакторе:

sudo EDITOR=nvim visudo

и добавить в конец строку:

blxnk ALL=(ALL) NOPASSWD: /usr/bin/systemctl restart tor.service

где blxnk, имя пользователя.

Перезапуск GNOME Shell

Можно использовать, если в gnome появились какие то артефакты или зависшие уведомления. Имейте в виду, что все ваши окна и рабочий стол перезагрузятся.

killall gnome-shell

Запуск установщика системы из терминала

Для запуска установщика Fedora в режиме live среды, когда графический установщик не запускается автоматически.

sudo dbus-launch liveinst

Получение текущей раскладки клавиатуры из терминала

Команда позволяет получить список источников ввода (раскладок клавиатуры) в GNOME, отсортированный по наиболее недавно использованным. Это значение представляет собой список раскладок, которые вы использовали недавно, где первая раскладка в списке — это текущая активная раскладка.

gsettings get org.gnome.desktop.input-sources mru-sources

Вывод терминала:

[('xkb', 'ru'), ('xkb', 'us')]
